# Security_Policy_Dev
SECURITY POLICY DEVELOPMENT PROJECT REPORT

# Overview

This project focuses on developing core cybersecurity policies for Cybertech, leveraging best practices from the SANS Institute Security Policy Templates.

It establishes a structured approach to securing organizational systems by defining clear rules, responsibilities, and controls around user behavior and access management.
# Objectives

✔ Standardize security policies across the organization
✔ Align with globally recognized cybersecurity frameworks
✔ Reduce human-related security risks
✔ Strengthen identity and access management
✔ Promote security awareness culture
# Policies Developed
# 1. Acceptable Use Policy (AUP)

Defines how employees are permitted to use Cybertech’s IT resources.

Highlights:

* Responsible use of company devices
* Internet & email usage controls
* Prohibited activities (unauthorized access, piracy, misuse)
* Monitoring and compliance enforcement
# 2. Password Policy

Ensures strong authentication and account protection.

Highlights:

* Minimum complexity requirements
* Password expiration & reuse limits
* Multi-Factor Authentication (MFA)
* Account lockout protection
# 3. Access Management Policy

Controls and governs access to systems and sensitive data.

Highlights:

* Role-Based Access Control (RBAC)
* Principle of Least Privilege (PoLP)
* User provisioning & de-provisioning
* Access reviews and audits
# Framework

This project is built using:

* SANS Security Policy Templates
* Industry-standard security governance practices
* Core principles:
Confidentiality
Integrity
Availability (CIA Triad)
